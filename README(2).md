# Моделирование эффективности турнирных сеток

Репозиторий содержит программную реализацию вычислительных экспериментов для выпускной квалификационной работы «Модель оценки эффективности турнирных сеток».

## Постановка задачи

Входными данными служит матрица попарных вероятностей побед

\[
P=(p_{ij}), \qquad p_{ij}+p_{ji}=1,
\]

где `p[i, j]` задаёт вероятность победы участника `i` над участником `j`.

По матрице вычисляется предварительный показатель команды

\[
s_i=\sum_j p_{ij}.
\]

Он интерпретируется как ожидаемое число побед в обобщённом круговом турнире и используется для задания целевого порядка участников. Для каждого допустимого посева рассчитывается вероятность того, что команды достигнут соответствующих целевых стадий. Лучшим считается посев с максимальным значением целевой функции.

## Реализованные форматы

- **Single Elimination**: турнир с выбыванием после первого поражения. Вероятности прохождения раундов вычисляются динамическим программированием.
- **Double Elimination**: турнир с выбыванием после второго поражения. Вероятности вычисляются обходом дерева возможных исходов по явному графу матчей.

## Возможности

Код позволяет:

- задавать или генерировать матрицы попарных вероятностей;
- вычислять предварительный рейтинг участников;
- строить классический посев;
- считать точное значение целевой функции для фиксированного посева;
- перебирать существенно различные посевы с учётом симметрий сетки;
- находить оптимальный посев;
- сравнивать классический посев с минимумом и максимумом целевой функции;
- моделировать отдельные реализации турнира;
- проводить эксперименты на случайных и транзитивных матрицах.

## Программы

Для каждого из двух форматов используются программы следующих типов:

- `subgrid.py` - определение множества возможных соперников;
- `Просчет вероятностей.py` - точное вычисление вероятностей исходов;
- `Симуляция.py` - моделирование одного турнира;
- `Оптимизатор.py` - полный перебор посевов и поиск оптимального;
- `Сравнение результатов.py` - сравнение классического и оптимального посевов;
- `Специальные матрицы.py` - эксперименты на транзитивных epsilon-матрицах.

Названия и расположение файлов в репозитории могут отличаться. При запуске используйте фактический путь к соответствующему скрипту.

## Требования

- Python 3.10 или новее;
- NumPy.

Установка зависимости:

```bash
python -m pip install numpy
```

## Запуск

Пример запуска скрипта из каталога, в котором он расположен:

```bash
python "Оптимизатор.py"
```

Если код разделён по каталогам форматов, сначала перейдите в нужный каталог либо укажите полный путь к файлу.

Большинство программ запрашивает параметры через стандартный ввод. В зависимости от выбранного скрипта могут потребоваться:

- параметр `N`, для которого число участников равно `2**N`;
- число генерируемых матриц;
- элементы матрицы вероятностей;
- значение `epsilon` для специальной транзитивной модели.

## Интерпретация результатов

Для классического посева используется относительная точность

\[
A(P)=\frac{F(P,\sigma_{classic})-F_{min}(P)}{F_{max}(P)-F_{min}(P)}.
\]

Значение `1` означает, что классический посев оптимален для данной матрицы. Значение `0` означает, что его целевая функция совпадает с минимальной среди рассмотренных посевов.

## Ограничения

Полный перебор быстро становится вычислительно дорогим при увеличении числа участников. Реализация предназначена прежде всего для сеток на 4 и 8 участников. Матрица вероятностей считается заданной; задача её оценки по данным матчей в репозитории не решается. Транзитивность вероятностей также не предполагается.
